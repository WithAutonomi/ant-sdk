# Autonomi FFI Bindings

UniFFI-based foreign function interface bindings for the Autonomi network. Wraps the `autonomi` Rust crate directly (no daemon, direct network access) and auto-generates language bindings.

## Supported Languages

- **C#** - Generated via [`uniffi-bindgen-cs`](https://github.com/NordSecurity/uniffi-bindgen-cs) (v0.10.0+v0.29.4)
- **Kotlin** - Generated via `uniffi-bindgen` (v0.29.4) with JNA
- **Swift** - Generated via `uniffi-bindgen` (v0.29.4)

## How It Works

1. **Rust proc macros** (`#[derive(uniffi::Record)]`, `#[uniffi::export]`, etc.) annotate wrapper types
2. `uniffi::setup_scaffolding!()` generates the FFI scaffolding at build time
3. **`uniffi-bindgen-cs`** reads metadata from the compiled native library and generates complete C# bindings (P/Invoke, async `Task<T>`, types, error handling)
4. **`uniffi-bindgen`** generates Kotlin bindings (JNA + coroutines) and Swift bindings

No UDL files needed — everything is driven by Rust proc macros in **library mode**.

## Build

### Prerequisites

- Rust toolchain (1.82+)
- .NET 8.0 SDK (for C# bindings)
- JDK 17+ and Gradle (for Kotlin bindings)
- Swift 5.9+ / Xcode 15+ (for Swift bindings, macOS/Linux only)
- `uniffi-bindgen-cs`: `cargo install uniffi-bindgen-cs --version "0.10.0+v0.29.4"`
- `uniffi-bindgen`: `cargo install uniffi-bindgen-cli --version "0.29.4"`

### Quick Build

```bash
# Linux/macOS
./scripts/build.sh

# Windows
.\scripts\build.ps1
```

### Manual Steps

```bash
# Step 1: Build Rust FFI library
cd rust && cargo build --release -p ant-ffi

# Step 2a: Generate C# bindings
uniffi-bindgen-cs --library target/release/ant_ffi.dll --out-dir ../csharp/AntFfi/Generated/

# Step 2b: Generate Kotlin bindings
uniffi-bindgen generate --library target/release/ant_ffi.dll --language kotlin --out-dir ../kotlin/AntFfi/Generated/

# Step 2c: Generate Swift bindings
uniffi-bindgen generate --library target/release/ant_ffi.dll --language swift --out-dir ../swift/AntFfi/Generated/

# Step 3: Build .NET solution
cd ../csharp && dotnet build AntFfi.sln

# Step 4: Build Kotlin project
cd ../kotlin && gradle build
```

## API Surface

The Autonomi network is **immutable, content-addressed storage** (chunks →
self-encrypted data → files) plus an EVM payment layer. There are no mutable
data types (registers/pointers/scratchpad/vault/GraphEntry) and no network
directory/archive primitive — those are separate products and are not exposed.

Most methods are on the `Client` object (names are the Rust names; UniFFI
renders them camelCase in Swift/Kotlin, e.g. `chunk_put` → `chunkPut`):

| Domain | Methods |
|---|---|
| **Connect** | `connect_default`, `connect_default_with_wallet`, `connect_default_for_external_signer` (production network, vendored peers), `connect_local`, `connect`, `connect_with_wallet`, `connect_from_devnet_manifest`, `connect_from_devnet_manifest_external_signer`, `connect_for_external_signer` — all take an optional `data_dir` (required on Android: pass the app's files dir) |
| **Chunks** | `chunk_put`, `chunk_get`, `chunk_exists` |
| **Data (bytes)** | `data_put_public`, `data_get_public`, `data_put_private`, `data_get_private` |
| **Files** | `file_upload_public`(`_with_progress`), `file_upload_private`(`_with_progress`), `file_download_public`, `file_download_private`, `download_public_to_file`, `download_private_to_file`, `estimate_file_cost`(`_with_progress`) |
| **Data maps** | `data_map_store`, `data_map_fetch` |
| **External signer** | `prepare_data_upload`, `prepare_file_upload`, `prepare_file_upload_with_progress`, `payment_transactions`, `finalize_upload`(`_with_progress`), `finalize_upload_merkle`(`_with_progress`), `cancel_upload` |
| **Wallet** | `wallet_approve`; `Wallet` object: `from_private_key`, `address`, `balance_of_tokens`, `balance_of_gas_tokens` |
| **Free fns** | `network_info`, `wait_for_receipt`, `merkle_winner_pool_hash`, `ant_ffi_version` |

## Type Mapping

| Autonomi Type | FFI Representation | Reason |
|---|---|---|
| `Bytes` | `Vec<u8>` | UniFFI native support |
| `XorName` | `Vec<u8>` (32 bytes) | Validate at boundary |
| `Multiaddr` | `String` | Parse on Rust side |
| `AttoTokens` / `Amount` | `String` | Too large for u64 |
| `PathBuf` | `String` | Convert at boundary |
| `PublicKey` / `SecretKey` | `uniffi::Object` wrapper | Hex encode/decode |
| Tuples `(cost, addr)` | `uniffi::Record` structs | UniFFI can't do tuples |
| Complex errors | Flattened string-based enums | Keep FFI surface clean |

## Async

All async Client methods use `#[uniffi::export(async_runtime = "tokio")]`. The generated C# bindings expose them as `Task<T>` methods with proper async/await support. The generated Kotlin bindings expose them as `suspend` functions for use with Kotlin coroutines. The generated Swift bindings expose them as `async throws` functions for use with Swift concurrency.

## Security Notes

- **Private key zeroing**: Private keys passed to the FFI layer are zeroed from memory (via the `zeroize` crate) immediately after use. The original string content is overwritten before deallocation.
- **Hardware wallets**: For high-security environments, consider hardware wallet integration via the external signer flow (`prepare_data_upload` / `finalize_upload`) rather than passing raw private keys.
- **No key persistence**: The FFI layer does not persist private keys to disk. Keys exist only in memory for the duration of wallet construction.
